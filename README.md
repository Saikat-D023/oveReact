# oveReact

A custom-built, lightweight React clone written entirely from scratch in TypeScript. I created this to deeply understand how React's core concepts work under the hood!

Unlike basic brute-force clones, **oveReact** features a true Virtual DOM diffing engine, native HTML DOM property bindings (including Refs), and a robust Hooks system that faithfully mirrors standard React behavior.

## ✨ Features

- **Virtual DOM Engine:** Custom `createElement` (for JSX transpilation) and a recursive `renderNode` engine.
- **Smart Diffing:** An `updateElement` algorithm that compares old and new Virtual DOMs, updating *only* the specific physical DOM nodes that changed, avoiding unnecessary full-page repaints.
- **Native DOM Refs:** Deep integration that intercepts the `ref` prop and natively attaches physical HTML elements so you can execute imperative DOM methods (like `.focus()`).
- **Hooks System:**
  - `useState`: Stateful memory using pointer indexes, enabling targeted re-renders via `triggerRerender`.
  - `useEffect`: Dependency tracking (`[]`) that utilizes `setTimeout` to guarantee execution *after* the DOM paints.
  - `useRef`: Memory persistence across renders using standard React API (mutable `{ current }` objects).
- **CLI Scaffolding:** A built-in CLI tool (`cli.ts`) that automatically generates a new project template for users, completely mimicking tools like `create-react-app`.

## 🚀 Setup & Running

This project uses Bun's built-in bundler to compile the TypeScript/JSX source code (`src/app.tsx`) into a browser-ready JavaScript bundle (`dist/bundle.js`).

### 1. Install Dependencies
```bash
bun install
```

### 2. Start the Development Server
This will watch your `src/` files and automatically rebuild the `dist/bundle.js` bundle whenever you hit save.
```bash
bun run dev
```

### 3. View the App
Open a second terminal window and serve the `index.html` file using any static server. For example:
```bash
npx serve .
```
Navigate to `http://localhost:3000` (or whichever port is provided) to see the interactive Demo Application running on the custom framework!

## 🛠️ Testing the CLI Scaffold Tool

You can generate a fresh project powered by this framework using the custom CLI script.

To test it locally:
```bash
bun cli.ts my-new-app
```
This will instantly create a folder named `my-new-app` equipped with a full starter template!

*(Eventually, this can be published to NPM so anyone can run `npx overeact <project-name>`!)*

## 📁 Project Structure

- `index.html`: The main HTML file that hosts the application and loads the bundled script.
- `cli.ts`: The scaffolding executable for creating new projects.
- `src/`: The core framework and application logic.
  - `app.tsx`: The Demo Application containing interactive examples of state, effects, and refs.
  - `vdom.ts`: Virtual DOM creation (`createElement`), DOM rendering (`renderNode`), and smart diffing (`updateElement`).
  - `hooks.ts`: Implementation of global state memory and standard hooks (`useState`, `useEffect`, `useRef`).
  - `renderer.ts`: Component mounting logic and the `triggerRerender` loop.
  - `index.ts`: The framework's public API that exports all utilities to the end-user.
- `dist/`: Contains the bundled output.
  - `bundle.js`: The browser-compatible JavaScript file generated by the `bun run dev` build process.
